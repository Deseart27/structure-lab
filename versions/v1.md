# V1 — Current FullEnrich App (Baseline)

## Navigation

Search | Enrichment | Reverse Email Lookup (New) | API Console | MCP & Integrations

## Structure

### Search
- People and Companies tabs
- Results table with checkboxes
- Actions: Export, Add to list, Enrich

### Enrichment
- "New Enrichment" launcher: CSV or Manual input
- Enrichment type selector: Phone + Email / Phone / Email (radio-style)
- "Contact List" section below with tabs: All Lists / Contact Lists / CSV Files
- Each enrichment row shows: name, type, date, found/total, status (Completed/Running)
- Results and lists are mixed together in the same section

### Reverse Email Lookup
- Separate top-level nav item (badge "New")
- Standalone page, not a sub-section of Enrichment

### API Console
- Separate top-level nav item

### MCP & Integrations
- Separate top-level nav item

## Key characteristics
- Enrichment section is both the launcher AND the result viewer — everything lives here
- No concept of "Lists" as a separate section — contacts are inside enrichment
- Reverse Lookup is disconnected from Enrichment (separate nav item)
- Export available from Search
- No sub-navigation bar
- No "Push to CRM" actions on search results

## Problems this structure creates
- Enrichment section becomes a dumping ground: launcher + history + contact browser
- Reverse Lookup feels orphaned — same category as enrichment but separate nav
- No unified view of all your contacts across enrichment types
- API Console as a top-level nav item takes space for a niche feature
