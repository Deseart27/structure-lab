# Version 4 — Find, Enrich, Push

## Philosophy

V4 is designed around 3 core principles derived from 8 real user pain points:

1. **Unified Lists** — All lists live in one place, regardless of source (search, CSV enrichment, reverse lookup, manual). No fragmentation.
2. **Universal Export** — Export/Push is a contextual action (popover), not a section. Available everywhere with the same UI: Push to CRM, Export CSV, Push to engagement tool.
3. **Optional Lists** — Lists are never a prerequisite. Enrichment jobs from CSV/manual are standalone objects that can be exported directly. Lists are optional organization.

## Navigation

| Section | Sub-nav | Description |
|---------|---------|-------------|
| Search | People \| Companies | Find B2B contacts and companies. Same as v2. |
| Enrichment | Emails & Phones \| Reverse Lookup \| CRM Enrichment | Enrich contacts with verified data. |
| Lists | *(none)* | All lists from all sources in one unified view. |
| Integrations | *(none)* | CRM and tool connections. |

## Key Behaviors

### Search → Enrich flow
- When enriching from search, a list is **always required** (contacts need to live somewhere).
- The system **preselects the last used list** or a default list to make it seamless.
- "Add to list" is available as a separate action (without enrichment).

### Enrichment section (CSV / Manual / List / Reverse / CRM)
- **"Pick from your lists"** is available as an input method — the user can create a list (from search, CSV import, or manually), then come back to enrichment and enrich that list.
- **No destination list selector** — when enriching from a list, the list already exists. For CSV/manual, enrichment jobs are standalone objects.
- Completed enrichments show: **"Add to list"** (optional) + **"Export"** (universal popover).
- Enrichment jobs are persistent — users can always come back and export results.

### Lists section
- **Unified view**: all lists from search, CSV enrichment, reverse lookup, manual creation.
- Each list shows a **source badge** (Search, CSV, Reverse, Manual).
- Actions per list: **Enrich** + **Export** (universal popover with Push to CRM, Export CSV, Push to engagement tool).
- **"New List"** button for creating empty lists.
- No sub-navigation (unlike v3 which has Lists | All Contacts).

### Universal Export popover
Same UI everywhere it appears:
- Push to CRM (HubSpot icon)
- Export CSV
- Push to engagement tool

## Pain Points Addressed

1. **"Where are my enriched contacts?"** → Enrichment jobs are persistent with Export action right there.
2. **"I can't find my lists"** → All lists in one place, no fragmentation.
3. **"I want to push to CRM but can't find the button"** → Export popover available on every output.
4. **"Lists from search and CSV are in different places"** → Unified lists with source badges.
5. **"I have to create a list before enriching CSV"** → No list required for CSV/manual enrichments.
6. **"I enriched from search but can't export"** → Export available on list view.
7. **"Push to HubSpot is hidden"** → Universal Export popover, always accessible.
8. **"Too many clicks to go from search to enriched + pushed"** → Seamless list preselection, Export everywhere.
