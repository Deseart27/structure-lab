# Version 5 — Two Bricks: Data Providing + Data Enrichment

## Philosophy

FullEnrich is NOT a contact database (like Lusha). It's a B2B data engine built on two core capabilities:

1. **Data Providing** — Find people and companies matching criteria (search, filters)
2. **Data Enrichment** — Take contacts (from any source) and find their emails, phones, verified data

These two bricks can be used independently or sequentially. Output (Export/Push) is always a contextual action, never a section.

## Three user personas / use cases

| # | User says | Flow | Bricks used |
|---|-----------|------|-------------|
| 1 | "I have contacts, enrich them" | Upload CSV / sync CRM / paste manually -> Enrichment -> Export | Enrichment only |
| 2 | "Find me people matching X" | Search -> Browse -> Export raw results | Data Providing only |
| 3 | "Find me people WITH their emails/phones" | Search -> Enrich -> Export | Both (highest value) |

Use case 3 is the golden path and highest revenue driver.

## Navigation

| Section | Description |
|---------|-------------|
| Search | Find people and companies. Browse results. Export raw OR enrich from here. |
| Enrichment | All enrichment jobs from all sources. The engine room. |
| Lists | Optional grouping of contacts across jobs. Organizational layer, never a prerequisite. |
| Integrations | CRM connections, engagement tools, export settings. |

No sub-navigation on Lists. Sub-nav on Search (People | Companies) and Enrichment (Emails & Phones | Reverse Lookup | CRM Enrichment).

## Key Behaviors

### Search section
- Browse results with basic info (name, title, company, LinkedIn, location)
- **Row-level "Enrich" button** — click on any individual contact to enrich inline. Email/phone appears in the row after processing.
- **Batch "Enrich" button** — select multiple contacts, click Enrich. Optional list picker in the modal.
- **Export raw** — export search results without enriching (Data Providing only use case)
- Contacts enriched from search without a list are automatically grouped into a session-based enrichment job (named after the search query)

### Enrichment section (the engine room)
- **Job-based view** — every enrichment from any source appears as a job
- Job sources: search session, CSV upload, reverse lookup, CRM sync, manual input
- Each job shows: source, contact count, progress (%), status, date
- Click into a job to see contacts + enrichment results
- Actions per job: Export (universal popover) + Add to list (optional)
- Jobs from casual search enrichment are auto-created and named after the search query (e.g. "VP Sales, SaaS, France — 3 contacts")

### Lists section
- Optional organizational layer — never required for enrichment or export
- Users can create lists manually, from search results ("Add to list"), or from enrichment job results
- Each list aggregates contacts from potentially multiple jobs/sources
- Actions per list: Enrich + Export (universal popover)
- Source badges on contacts within lists (Search, CSV, Reverse, CRM, Manual)

### Universal Export popover
Same UI everywhere (on jobs, on lists, on search results):
- Push to CRM (HubSpot, Salesforce)
- Export CSV
- Push to engagement tool (Outreach, Apollo, etc.)

## Search -> Enrich flow (the golden path, use case 3)

### Casual browsing enrichment (one by one)
1. User searches "VP Sales, SaaS, France"
2. Browses results, clicks "Enrich" on a row
3. Row shows spinner, then email/phone appears inline
4. Does this a few more times across pages
5. Closes tab
6. Next day: goes to Enrichment -> sees job "VP Sales, SaaS, France (3 contacts)"
7. Opens job -> exports or adds to list

### Batch enrichment from search
1. User searches, selects 50 contacts
2. Clicks "Enrich" -> modal: "Enriching 50 contacts" with optional list picker
3. Job created in Enrichment section
4. User redirected to job page (or stays on search, toast notification)
5. Job shows progress -> done -> Export

### Search without enrichment (use case 2)
1. User searches "employees at Datadog"
2. Browses results
3. Exports raw CSV or adds to list for later

## Key differences from previous versions

| Aspect | V2/V4 | V5 |
|--------|-------|----|
| Enrichment from search | Requires a list | No list required — auto-creates session job |
| Where results live | In lists | In enrichment jobs (lists are optional grouping) |
| Central object | List | Job |
| Data providing (raw search) | Implicit, always tied to enrichment | Explicit, standalone value |
| "Where are my results?" | Check your lists | Check Enrichment section |
