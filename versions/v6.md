# Version 6 — Jobs + Parallel Lists

## Philosophy

V6 is built on two structural principles:

1. **Jobs are automatic containers.** Every credit spent creates a job entry. The enrichment job IS the container for results — no list required, no naming ceremony. Jobs are created automatically from search sessions, CSV uploads, reverse lookups, CRM syncs, and manual entries.
2. **Lists are explicit and optional.** Lists only exist by deliberate user action ("Add to list"). They are an organizational layer on top of jobs, never a prerequisite for enrichment or export.

Plus one key structural move vs all previous versions: **the Enrichment section IS the jobs table.** No sub-pages for Emails & Phones / Reverse / CRM. All input modes live inside one "New enrichment" launcher button. Like Linear: Issues is the noun, "New issue" is the button.

## Navigation

| Section | Sub-nav | Description |
|---------|---------|-------------|
| Search | People \| Companies | Find B2B contacts and companies. Inline enrich, batch enrich, export, push to CRM — all without leaving search. |
| Enrichment | *(none)* | All enrichment jobs from all sources in one table. The engine room. Global contact search bar. "New enrichment" launcher. |
| Lists | *(none)* | Optional organization. People lists and Company lists in one flat table. |
| Integrations | CRM \| Engagement \| API & MCP | CRM connection + push history, engagement tools, API keys and MCP. |

## Key Behaviors

### Search — People

- Results table with checkboxes + **row-level "Enrich" button** on every row
- **Inline enrichment (zero friction):** click Enrich on a row → spinner → email/phone appear inline in the row → toast "Contact enriched — logged in Enrichment"
- Behind the scenes: inline enrichments are grouped by search query as a job in the Enrichment section (e.g. "VP Sales · SaaS · France — 3 contacts")
- **Selection action bar** (appears when 1+ rows checked):
  - **Enrich** → modal with enrichment type checkboxes (Professional email, Phone, Personal email) + optional list picker (default: "No list"). No list forced.
  - **Add to list** → popover with existing people-lists. Duplicate detection: if a contact's LinkedIn URL already exists in the target list, it's skipped with a toast ("3 added, 1 duplicate skipped").
  - **Export** → Universal Export popover. **Push to HubSpot works directly from search with no list requirement** (Pain 1 solved).
- Search query chip displayed at top ("VP Sales · SaaS · France")

### Search — Companies

- Results table with clickable company names → **Company Profile** page
- **Funding badge** column: locked (grey) or unlocked (green) per company
- Selection actions: Add to list (company lists only), Export (Universal Export popover), Find employees

### Company Profile (new)

- Header: company name, domain, industry, headcount, location
- **Bundle cards** (Crunchbase-powered data):
  - **Last Funding** — locked: blurred skeleton + "Unlock — 2 credits". Unlocked: round, amount, date, total raised, investors list.
  - **Financials** — locked: blurred skeleton + "Unlock — 3 credits". Unlocked: revenue range, growth, employees trend.
  - **Technographics** — "Coming soon" (greyed)
  - **Intents** — "Coming soon" (greyed)
  - **News** — "Coming soon" (greyed)
- Unlock click → spinner → card fills with data → toast "Funding unlocked — cached for your workspace"
- Unlocked bundles are cached workspace-wide and appear as extra columns in exports

### Enrichment — the engine room

- **No sub-navigation.** The section IS a jobs table.
- **Header:** title, global contact search bar, "New enrichment" primary CTA button
- **Global contact search:** type 2+ chars → dropdown of matching contacts across ALL jobs and lists. Each result shows name, company, email, and **provenance chips** (colored badges: "CSV — leads_q3.csv", "List: Q3 Targets", "Search — VP Sales SaaS France"). One contact can appear in multiple jobs → multiple chips shown. Click → navigate to containing job detail.
- **Filters:** by Source (All / CSV / Manual / Reverse / CRM / Search) and by Status (All / Running / Completed)
- **Jobs table columns:** Source (icon + label), Name (editable, linked to detail), Contacts, Progress (bar + %), Status (badge: Running / Queued / Completed / Completed with errors), Date, Actions (Export popover, Add to list, Open)
- **Pinned running job:** after launching an enrichment, the new job appears at the top with a highlighted card and the text: "Enrichment runs in the background — you can safely leave this page." Progress animates to completion.

### "New enrichment" launcher

A modal with a 2-step flow:

**Step 1 — "What do you have?"** Five input-mode cards:
- Upload a CSV ("up to 64,000 contacts")
- Add manually ("name + company or LinkedIn URL")
- Emails to reverse ("paste emails, get full profiles") — "New" badge
- From my CRM ("enrich HubSpot lists")
- From a list ("enrich an existing FullEnrich list")

**Step 2 — Configure:** enrichment type checkboxes + mode-specific input (dropzone, textarea, list picker, etc.). **No destination list selector anywhere** — reinforces Rule 1: every enrichment creates a job, not a list.

Launch → close modal → land on jobs table with pinned progressing job → toast "Enrichment started"

### Job detail

- Header: source icon, editable name, status badge, progress, date
- Actions: Export (Universal Export popover), Add to list (optional), Back
- **Contact table:** Name, Company, Title, Professional email + status chip, Phone, Personal email, row Status
- **Email status chips:**
  - Valid (green)
  - Catch-all (yellow)
  - **Found but invalid (orange)** — explicitly NOT "Not found" (Pain 4 fix)
  - Not found (grey)
  - Pending (blue)
- Per-row: Add to list action

### Lists

- **No sub-navigation.** One flat table of ALL lists — people AND company lists together.
- **Type badge** per list: People (violet) / Company (blue)
- **Source badges** per list: multiple badges showing where contacts came from (Search, CSV, Reverse, Manual, Job)
- **"New list" button** → modal with name + type selector (People list / Company list)
- Actions per list: **Enrich** (people lists only — launches a job, results flow back into the list) + **Export** (Universal Export popover)
- List detail: contact/company table with provenance chips ("In job: leads_q3.csv"), remove action

### Integrations

Sub-nav: **CRM | Engagement | API & MCP**

**CRM tab:**
- HubSpot connected card (one CRM per workspace), "Switch CRM" option
- **Recent Pushes table:** date, source badge (Search / Job / List), source name, contacts count, status badge (Completed / Syncing / Failed), row actions: "Enrich these contacts" and "Save to list" (retroactive — both create new objects)
- Every Push to HubSpot action anywhere in the app creates a trace here
- Note: "Automated CRM enrichment rules (filters + scheduling) will be configured here; each run appears as a job in Enrichment."

**Engagement tab:** Grid of connect cards (Lemlist, Smartlead, Instantly, Outreach, Salesloft, HeyReach, etc.)

**API & MCP tab:** API key card + MCP server card

### Universal Export popover (shared component)

Same UI component everywhere (search selection, job row, job detail, list row, list detail, company profile):
- **Contacts:** All contacts / Enriched only (radio)
- **Data to include:** Contact info (always), Emails & Phones, Funding data (lock + credits hint), Financials (lock + credits hint)
- **Destinations:** Push to HubSpot / Export CSV / Push to engagement tool (sub-menu: Lemlist, Smartlead, Instantly, Outreach, Salesloft, HeyReach)
- Push to HubSpot → toast: "Pushing X contacts to HubSpot — contacts will be created or updated (3-way dedup). Trace available in Integrations > CRM."

## Pain Points Addressed

1. **"I can't push search results to CRM"** → Export popover with Push to HubSpot available directly from search selection. No list required. Trace lands in Integrations > CRM. (Most documented complaint — 8+ Intercom tickets.)
2. **"Search lists and Enrichment lists are separate"** → One unified Lists section with mixed source badges. All lists in one table.
3. **"I don't know where to go first"** → Clear structure: Search to find, Enrichment to process, Lists to organize. Enrichment is always the answer to "where are my results?"
4. **"Invalid emails shown as Not found" / "No reassurance for long jobs"** → "Found but invalid" email status chip (orange). Pinned running job with "runs in the background" message.
5. **"I don't understand what Push to CRM does"** → Export popover toast: "contacts will be created or updated (3-way dedup)." Recent Pushes table in CRM tab shows full history.
6. **"Some sections can push, others can't"** → Universal Export popover renders identically in all 7+ locations.
7. **"I have to create a list before doing anything"** → Lists are never required. Every enrichment creates a job. Export from jobs directly.
8. **"Where did my enriched contacts go?"** → Always in Enrichment. Inline search enrichments auto-grouped by query. Global contact search finds any contact across all jobs.

## Key Differences from Previous Versions

| Aspect | V2/V4 | V6 |
|--------|-------|----|
| Enrichment section | Sub-pages (Emails & Phones / Reverse / CRM) | One jobs table + "New enrichment" launcher |
| Central object | List (V2) / mixed (V4) | Job |
| Enrichment from search | Requires a list (V2/V4) | No list required — auto-creates session job |
| Inline enrich from search | Not available | Row-level Enrich button with inline results |
| Push to CRM from search | V2: yes / V4: via Export popover | Yes, via Export popover, with push trace |
| Push history | Not tracked | Recent Pushes table in Integrations > CRM |
| Company data bundles | Not available | Funding + Financials cards, unlockable per company |
| Global contact search | Not available | Search bar in Enrichment, cross-job provenance |
| Email status "Found but invalid" | Shown as "Not found" | Explicit orange chip |
| Enrichment reassurance | None | Pinned job card with background message |
| Lists | Required for export (V2) / optional (V4) | Fully optional, typed (People / Company) |
| Integrations | Flat page | Sub-nav: CRM (with push history) / Engagement / API & MCP |

## Design Rules (encoded in behavior)

1. **Enrichment never requires or creates a list.** Every credit spent creates a job entry. The job is the container.
2. **Lists only exist by explicit user action.** "Add to list" is always optional and always available post-hoc.
3. **No persistent contact entity.** Contacts live inside jobs and lists. No global contact database. Dedup only at list insertion (same LinkedIn URL → skipped).
4. **Universal Export popover.** Same component, same options, everywhere.
5. **Every push creates a trace.** All Push to CRM actions land in Integrations > CRM > Recent Pushes.
